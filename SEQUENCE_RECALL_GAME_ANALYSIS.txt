1. OVERVIEW
========================================================================
Sequence Recall is a Simon-Says style tile memory game. Colored tiles 
flash in a sequence, and the player must repeat the pattern in order. 
Each successful round adds one more tile to the sequence. The game 
features a session timer (countdown), multiple difficulty levels, 
demo/tutorial mode, and wrong-move handling strategies.

Genre: Memory / Sequence / Pattern Recall
Core loop: Watch tiles flash → Repeat in order → Sequence grows → Survive

4. GAME MECHANICS
========================================================================

4.1 PHASES (GAME_PHASE enum)
- loading: Initial load
- ready: Lobby/instructions shown
- tutorial: Tutorial overlay
- showing-sequence: Tiles flash in order (playback)
- awaiting-input: Player taps tiles in sequence
- round-success: Round complete, advancing
- round-failure: Wrong tile, handling failure
- game-over: Game ended (timer/wrong move)
- session-complete: All rounds completed

4.2 TILE SYSTEM
- boxCount: configurable 2-6 tiles
- Tiles numbered 1..boxCount
- 4 colors: green(1), red(2), blue(3), yellow(4) + extras for 5/6
- Each tile plays unique sound (piano or retro audio pack)
- Sequence no triple-repeat constraint

4.3 SEQUENCE GENERATION
- generateSequence(length, boxCount, rng)
- No three consecutive same tiles
- Sequence grows each round: revealedSequence = sequence[0..currentLength]
- initialSequenceLength: configurable start (e.g., 2 tiles)

4.4 PLAYBACK
- tileFlashMs: duration of each tile highlight
- tileGapMs: gap between tile flashes
- speedMultiplier: accelerates over time
- inputGlowMs: visual feedback on player tap

4.5 SCORING
- baseScorePerSound × sequenceLength per round
- streakMultiplier: bonus for consecutive correct rounds
- perfectRoundBonus: bonus for no mistakes
- Score calculated per validated round

4.6 WRONG MOVE HANDLING (4 strategies)
- GAME_END (1): Game over immediately
- PLAY_AGAIN (2): Replay same round
- PREV_SEQUENCE (3): Go back one step
- NEXT_SEQUENCE (4): Move forward (skip)

4.7 TIMER
- sessionTimerSeconds: whole-session countdown
- endTime from server (ms epoch)
- Timer expiry → game over

4.8 LIVES
- initialLives: configurable (usually 1-3)
- Wrong move costs a life (strategy-dependent)

4.9 DEFAULT CONFIG
- boxCount: 4 tiles
- initialLives: 1-3
- sessionTimerSeconds: 60+ (server-configured)
- initialSequenceLength: 2
- turnLimit: 0 = unlimited
- baseScorePerSound: 10

7. API ENDPOINTS (socket-based)
========================================================================
- game:start → { gameSessionId, endTime, isResumed, maxRounds, serverConfig }
- game:new_round → { sequence data, endTime }
- game:validate → { isCorrect, completedRound, wonGame, state }
- game:game_over → { finalScore, bonus, completedRounds, reason }
- game:time_sync → { endTime, serverTime }
