3. GAME MECHANICS
================================================================================

Core Concept: Classic sliding tile puzzle with image fragments
- Grid: configurable (e.g. 3×3, 4×4, 5×5 etc.)
- One blank tile (value -1), slide tiles into blank position
- Goal: reassemble the full image from shuffled pieces
- Multiple rounds per session (totalRounds from server)

Game Phases (GamePhase):
- idle: waiting to start
- loading: connecting/fetching data
- memorizing: shows solved image with countdown timer (displayTime from server)
- shuffle-animation: gather-to-center then scatter animation
- playing: player slides tiles
- board-clearing: transitioning between rounds
- loading-next: fetching next round's board
- revealing: solved board reveal animation
- calculating_result: waiting for server score
- completed: game finished (success/failure)
- expired: timer ran out
- error: connection/API error

Shuffle Animation Sequence:
1. Start with solved positions (750ms delay)
2. Gather all tiles to center (550ms transition + 120ms settle)
3. Scatter to shuffled positions (550ms + 200ms settle)
4. Start playing

Memorize Phase:
- Shows full image overlay with countdown bar
- displayTime from server (seconds)
- Image preloads before countdown starts
- Fades out when complete

Tile Movement:
- Click/tap a tile adjacent to blank → tile slides into blank position
- Invalid moves get "wrong step" sound
- Valid moves get "whoosh" sound
- Move submission: batched to server with timestamps

Solving Detection:
- isSolvedBoard() checks if pieces match sequential order
- On solve: reveal overlay, success sound, level-complete burst effect
- Round score calculated server-side

================================================================================
4. SCORING & PROGRESSION
================================================================================

- Multiple rounds per session (totalRounds)
- Timer: session-level countdown (expiryAt from server)
- Each round has: roundNumber, gridSize, fullImageUrl, displayTime
- RoundScore: { roundNumber, score }
- Final scoreboard: { rounds, timeBonus, totalScore }
- Results: GameSuccessScreen (all rounds) or GameFailureScreen (expired)

- GameFailureScreen: stage, score, completedGames, totalGames, onContinue

================================================================================
6. SOUND SYSTEM
================================================================================

useSoundSystem hook:
- Background music: /sounds/background_sound.mp3 (loop, 35% base vol)
- Tap: /sounds/tap_sound.mp3 (70%)
- Whoosh: /sounds/whoosh_sound.mp3 (50%) - valid tile move
- Wrong step: /sounds/wrong_step.mp3 (80%) - invalid move
- Success: /sounds/success_sound.mp3 (100%) - board solved

Volume controls:
- musicMultiplier (0-1) saved to localStorage 'game:musicvol'
- sfxMultiplier (0-1) saved to localStorage 'game:sfxvol'
- SoundSettings component with dual sliders

================================================================================
7. DEMO FLOW
================================================================================

- Fetched from API (getDemo)
- DemoLevels: array of {levelName, boards[]}
- DemoBoard: {id, gridSize, fullImageUrl, displayTime, initialPieces}
- Same memorize → shuffle → play flow as real game
- Completion modal on solve
- No timer, no scoring
- Can be toggled with "Learn How To Play" from start screen

================================================================================
8. SESSION PERSISTENCE
================================================================================

- PersistedLiveSession in localStorage
- Fields: token, sessionId, expiryAt, startedAt, clientStartedAtMs, totalRounds,
  currentRound, board, pieces, phase, capturedAtMs
- Auto-resumes on page refresh if valid session exists
- Server-anchored timestamps for move timing accuracy

10. API INTEGRATION
================================================================================

Auth: useAuth hook (authenticate, isLoading, isAuthenticated, error)
Game API (gameApi.ts):
- gameStart() → GameSession
- getNextBoard() → GameBoard
- getGameStatus() → GameStatusResponse
- endGame() → EndGameResponse
- getDemo() → DemoResponse
- submitMoves(moves) → SubmitMovesResponse
- endBoard() → EndBoardResponse

Move Submission:
- Batched via useMoveSubmission hook
- Each move: {slot, clickedAt (ISO), moveId (UUID)}
- Server-anchored timestamps via getServerAnchoredTimestamp

Board Prefetch:
- useBoardPrefetch hook for preloading next board
