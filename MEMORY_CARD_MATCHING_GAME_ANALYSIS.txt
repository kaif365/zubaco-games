3. GAME MECHANICS
========================================================================
Game States (GAME_STATES):
- LOADING → Fetching session/config from API
- PREVIEW → Cards shown face-up briefly (previewDurationSeconds per level)
- PLAYING → Player flips cards to find matches
- CHECKING_MATCH → Two cards flipped, checking if they match
- LEVEL_TRANSITION → Level complete, transitioning to next
- FINISHED → All levels complete or time ran out

Core Gameplay:
- Grid of cards (variable rows × columns per level)
- Cards have pairs (pairId) — flip 2 cards, if same pairId = match
- Multi-level: complete all levels within global time limit
- Time syncs with server (useTimeSync hook)
- Timer counts DOWN from gameTimeLimitSeconds
- Flip animation: 400ms (FLIP_ANIMATION_DURATION_MS)
- Level complete celebration: 800ms
- Round transition delay: 800ms

Card Content Types:
- symbol (emoji/unicode), image (URL), color, wordImage
- 24 built-in symbols (SYMBOLS array)
- 12 jewel palette colors

Scoring (from backend):
- Based on: speed, accuracy (fewer mismatches), combo streak
- Server calculates final score on game-over
