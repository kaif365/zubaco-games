1. GAME CONCEPT & GENRE
================================================================================

Genre: Real-Time Path Building / Node Connection / Graph Puzzle
Core Mechanic: Nodes appear progressively on a canvas over time. Player clicks
two nodes to connect them with an edge. Goal is to build the most efficient
route (shortest total path length) connecting as many nodes as possible before
the timer runs out.

Key Characteristics:
- Real-time pressure: nodes appear at intervals (every 2 seconds)
- Time-limited gameplay (60 seconds default)
- Strategic decision-making: which nodes to connect for efficiency
- Path efficiency scoring (optimal vs actual distance ratio)
- Click-two-nodes interaction (select first → select second → edge drawn)
- SVG-based canvas rendering with animated node appearance
- Seeded random generation (deterministic puzzles from server seed)

Game Mechanics:
- Nodes spawn at regular intervals (nodeIntervalMs = 2000ms)
- Player connects any two visible nodes by clicking them sequentially
- Score = path efficiency % + (nodes connected × 10)
- Path efficiency = (greedy nearest-neighbor optimal) / (actual path length) × 100
- Game ends when timer expires OR player submits early
- Backend validates edges & timing to prevent cheating

Unique Twist vs Static Puzzles:
- Unlike Flow Free or Connect Dots (static puzzles), this is LIVE/REAL-TIME
- Nodes appear over time — you must decide connections dynamically
- Similar to "Trail Making Test" (neuropsych assessment) but gamified
- Closest analogs: Mini Motorways (draw routes in real-time), Trail Making apps

3.3 Game Flow (Current):
─────────────────────────────────────────────────────────────────────────────────

1. App loads → InstructionScreen shown (idle state)
2. User clicks Start → API call to /game/start (gets seed + config)
3. Nodes begin appearing every 2s on SVG canvas
4. Player clicks node A → selected (highlighted amber)
5. Player clicks node B → edge drawn (blue line A→B)
6. Score updates in real-time (efficiency + node count)
7. Timer counts down from 60s
8. Timer hits 0 OR player clicks "Submit Early" → ended
9. GameOverScreen shows final score + Play Again

6. SCORING FORMULA
================================================================================

baseScore = pathEfficiency + (nodesConnected × 10)
comboBonus = streak × bonusPerLevel × comboMultiplier
timeBonus = (timeRemaining / totalTime) × 50
levelMultiplier = from level config (1.0 to 3.5)

finalScore = (baseScore + comboBonus + timeBonus) × levelMultiplier
