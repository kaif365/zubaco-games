14. [ ] APPLY TERRAFORM INFRASTRUCTURE
    Priority: CRITICAL
    Impact: Creates all AWS resources (DB, cache, containers, CDN, etc.)
    Steps:
      - Install Terraform CLI: https://terraform.io/downloads
      - cd c:\game\infrastructure\terraform\
      - Create terraform.tfvars with your values:
          aws_region          = "ap-south-1"
          environment         = "production"
          db_username         = "<choose-username>"
          db_password         = "<strong-password>"
          domain_name         = "yourdomain.com"
      - Run: terraform init
      - Run: terraform plan (review what will be created)
      - Run: terraform apply (type "yes" to confirm)
      - Save the outputs (RDS endpoint, Redis endpoint, ECR URLs, etc.)

18. [ ] DOMAIN & DNS SETUP
    Priority: HIGH
    Impact: Users can't reach your app without a domain
    Steps:
      - Buy/own a domain (e.g., zubaco.com)
      - In your domain registrar (or Route 53), create these records:
          api.zubaco.com        → CNAME → ALB DNS name (from terraform output)
          admin.zubaco.com      → CNAME → CloudFront distribution (admin frontend)
          admin-api.zubaco.com  → CNAME → ALB DNS name
          *.games.zubaco.com    → CNAME → CloudFront distribution (game frontends)
      - OR transfer domain to Route 53 (terraform already has Route 53 config)

19. [ ] SSL CERTIFICATES (ACM)
    Priority: HIGH
    Impact: HTTPS won't work without certs; browsers will block HTTP
    Steps:
      - Go to AWS Certificate Manager (ACM)
      - Request a public certificate for:
          *.zubaco.com
          zubaco.com
      - Validate via DNS (add the CNAME record ACM gives you)
      - Wait for validation (usually 5-30 minutes)
      - Attach certificate to:
          - ALB listener (HTTPS:443)
          - CloudFront distributions
      NOTE: If using terraform, it may have already created the cert.
            Check terraform output for certificate_arn.
