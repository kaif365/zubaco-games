1. GAME CONCEPT
========================================================================
Number Grid Sprint is a memory-recall speed game. A grid of cells
(e.g., 6×6) is presented. Numbers are hidden across the grid and
periodically "pulse" (reveal) in random groups for a brief moment.
The player must memorize the revealed values, then tap cells and
enter the correct number. The goal: fill the entire grid with correct
values before time runs out. Scoring rewards correctness + speed.

Core loop:
  1. Grid generated with unique values 1..N (deterministic via seed)
  2. Cells reveal in rotating batches (every hideIntervalMs, show
     ~25% of cells for revealDurationMs)
  3. Player taps a cell → number pad appears → enters guessed value
  4. Timer counts down → game ends when all cells filled OR time expires
  5. Score = correctCells × pointsPerCorrect + timeBonus

4. GAME MECHANICS DETAIL
========================================================================
Grid Generation:
  - Values 1..gridSize² placed via Fisher-Yates shuffle (seeded)
  - Each cell has { row, col, value }
  - Deterministic: same seed = same grid

Reveal Schedule:
  - Total intervals = timeLimitMs / hideIntervalMs
  - Each interval reveals ~25% of cells (totalCells/4)
  - Reveal lasts revealDurationMs, then hides
  - Offset seed (seed + 7919) for schedule

Player Interaction:
  - Tap cell → cell highlighted (blue border)
  - Number pad shows values 1..totalCells
  - Submit value → stored in playerAnswers Map
  - Can overwrite previously entered values
  - When all cells filled → auto-finish
  - "Submit Early" button available at any time

Scoring (Frontend):
  - correctCells × pointsPerCorrect (default 10)
  - timeBonus = floor(secondsRemaining × timeBonusMultiplier (default 2))
  - finalScore = cellScore + timeBonus

Default Config:
  - gridSize: 6 (36 cells)
  - revealDurationMs: 1000 (1s flash)
  - hideIntervalMs: 3000 (new batch every 3s)
  - timeLimitMs: 120000 (2 min)
  - pointsPerCorrect: 10
  - timeBonusMultiplier: 2
