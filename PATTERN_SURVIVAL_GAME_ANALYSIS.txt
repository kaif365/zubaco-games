1. GAME CONCEPT
========================================================================
Pattern Survival is a Simon-Says style pattern memory game. A 3×3 grid
of colored cells is displayed. Each round, the game flashes a sequence
of cells (the pattern). The player must then tap cells in the exact
same order. Each successful round adds one more step to the sequence.
The game continues until the player makes a mistake or time runs out.
This is a "survival" game — no fixed end, just keep going!

Core loop:
  1. Grid generated with random colors (6 colors, seeded)
  2. Sequence flashes one cell at a time (round N = N+1 cells)
  3. Player repeats the pattern by tapping cells in order
  4. Correct → next round (sequence grows by 1)
  5. Wrong tap → game over immediately
  6. Timer runs throughout; time out → game over
  7. Score = rounds × pointsPerRound

4. GAME MECHANICS DETAIL
========================================================================
Grid: 3×3 (9 cells), each colored randomly per round
Colors: red, green, blue, yellow, purple, orange (6 total)

Sequence Generation:
  - Per round: rng = mulberry32(seed + round × 7919)
  - Length = round + 1 (round 0 = 1 cell, round 5 = 6 cells)
  - Each cell index = floor(rng() × 9)

Cell Color Generation:
  - Per round: rng = mulberry32(seed + round × 3571)
  - Each of 9 cells assigned random color from pool

Showing Phase:
  - Cells flash one by one (500ms each, 200ms gap between)
  - Highlighted cell: scale 1.2 + ring-4 ring-white

Input Phase:
  - Player taps cells in sequence order
  - Each tap compared against expected sequence step
  - Wrong → immediate game over
  - All correct → advance to next round

Scoring:
  - pointsPerRound: 20 (per completed round)
  - perfectBonus: 10 (extra for no hesitation — unused currently)
  - score = rounds × 20
  - Submit: roundsReached, perfectRounds, clientScore

Timer:
  - Default: 300000ms (5 minutes)
  - Counts down continuously (across all rounds)
  - Display: MM:SS format, red when < 30s

7. TECHNICAL NOTES
========================================================================

- TailwindCSS class-based styling with COLOR_MAP Record
- No React Router — single component
- Survival mode: no win state, only failure (wrong tap or time out)
- Sequence grows indefinitely (round 0 = 1 step, round N = N+1 steps)
- Cell colors regenerate each round (visual variety)
- i18next for localization
- httpClient for API calls
- Save progress outbox (offline)
