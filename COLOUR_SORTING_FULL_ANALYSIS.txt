1. WHAT IS THIS GAME?
================================================================================

It's a "Ball Sort Puzzle" clone (also known as "Color Sort", "Tube Sort",
"Ball Sort"). The concept is identical to the popular mobile game:

  - You have tubes filled with coloured balls in random order
  - You move balls one at a time between tubes (top ball only)
  - A ball can only be placed on a same-colour ball OR into an empty tube
  - Goal: sort all balls so each tube has only one colour
  - Time-limited with scoring

This IS the same genre as "Ball Sort Puzzle" by IEC Global Pty Ltd (the viral
mobile game with 100M+ downloads). Same core mechanic, same rules.

5. SOUND SYSTEM
================================================================================

Sound IS implemented via Web Audio API (useAudio.ts):

Sounds available:
  - "tap" → Short 800Hz blip when selecting a tube
  - "correct" → C5+E5 rising tone on valid move
  - "incorrect" → 200Hz+150Hz sawtooth buzz on invalid move
  - "complete" → C5→E5→G5→C6 victory fanfare on puzzle solved
  - "countdown" → 440Hz square blip when timer low
  - "start" → 440Hz+880Hz rising on game start

PROBLEM: The useAudio hook EXISTS but is NOT called anywhere in the game
components! No component imports or uses it. Sound is dead code.

Settings persistence: localStorage key "zubaco_sound_enabled"

No background music. No volume control. No audio files (all procedural tones).

7. UI/UX ISSUES & BUGS
================================================================================

1. HARDCODED STAGE: Frontend always uses stageId "00000000-0000-0000-0000-000000000001"
   But the DB was seeded with stageId "colour-sort-stage-1". MISMATCH → 404 error!
   (Previously manually fixed by inserting a row with that UUID)

2. NO LEVEL PROGRESSION: Game plays same difficulty forever. No way to advance.

3. NO SOUND: useAudio hook exists but is never called from any component.

4. INVALID MOVE FEEDBACK: When you tap an invalid target, it just silently
   deselects or selects the new tube. No shake, no buzz, no visual feedback.

5. NO UNDO: Undo engine exists in backend but no button in UI.

6. SINGLE-BALL MOVES ONLY: You can't move a stack of same-colour balls
   (Ball Sort Puzzle mobile lets you move matching stacks in one tap).

7. NO RESTART OPTION: Once game starts, can't restart without finishing/timing out.

8. SMALL TUBES ON MOBILE: Tubes are only 48-56px wide. Hard to tap on small phones.

9. NO GAME STATE PERSISTENCE: If you refresh the page mid-game, everything is lost.

10. TIMER NOT PAUSED ON TAB SWITCH: Timer keeps running if user switches tabs.

11. RESULT SCREEN MINIMAL: Only shows "Moves" stat. Doesn't show time taken,
    efficiency breakdown, or comparison to best.

12. NO EMPTY STATE: If backend is down, user just sees "Failed to start" error.
    No retry mechanism shown.

10. SCORING SYSTEM DETAILS
================================================================================

Formula:
  finalScore = baseScore + efficiencyBonus + timeBonus

  baseScore = sortedTubes × pointsPerSortedTube (100-300 depending on stage)
  
  efficiencyBonus = max(0, (optimalMoves × 2 - actualMoves)) × 5
    where optimalMoves = colorCount × ballsPerTube
    (e.g., 3 colors × 4 balls = 12 optimal, so bonus if < 24 moves)

  timeBonus = floor(timeBonusMultiplier × (remainingTimeMs / timeLimitMs))
    (e.g., multiplier=10, 60s remaining/120s total → 5 bonus points)

With level multiplier: score × scoreMultiplier (1.0x to 3.5x)

Combo system (unused): +10 points × streak × multiplier per consecutive correct move
  Multiplier rises by 0.25 every 3 correct moves, caps at 4.0x

  - CheatFlag: detected anomalies (reason, severity)

================================================================================
12. SECURITY / ANTI-CHEAT MEASURES
================================================================================

  - Server replays all moves to verify final state
  - Score calculated server-side (client score is only for comparison)
  - Score divergence > 50 → cheat flag
  - Too many invalid moves (>3) → critical cheat flag
  - Client claims solved but server disagrees → critical flag
  - Session expiry enforced (endTime + 5s grace)
  - Double-submit prevented (atomic status update)
  - JWT required for all game endpoints
  - Optional AES-GCM request/response encryption
  - Idempotency keys on submissions

15. SUMMARY
================================================================================

The game is a Ball Sort Puzzle clone with:
  - Solid backend (NestJS, anti-cheat, provably fair)
  - Rich engine code (combos, boosters, achievements, daily challenges)
  - MINIMAL frontend (basic tap-to-move, timer, score — that's it)
  - Sound system written but NEVER CONNECTED
  - All advanced features exist as dead code in backend engines

It's like a car with a V8 engine but only 1st gear connected. The backend is
over-engineered and feature-rich. The frontend is a bare-bones MVP that doesn't
use 70% of what the backend offers.

To make this a SMOOTH, polished game comparable to Ball Sort Puzzle:
  1. Wire up the sound
  2. Add ball movement animation
  3. Add undo + restart
  4. Build level progression UI
  5. Add celebration effects
  6. Connect the booster/achievement systems
