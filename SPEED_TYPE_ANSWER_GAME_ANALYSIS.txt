3. GAME MECHANICS
================================================================================

Core Concept: Flash a question → Player types the answer from memory

Phases (type Phase):
- idle: Waiting to start (InstructionScreen shown)
- flash: Question displayed briefly (memorize)
- type: Player types answer within time limit
- result: Brief transition before next question
- finished: All questions done (ResultScreen shown)

Game Flow:
1. Player clicks "Start" on InstructionScreen
2. POST /game/start → receives gameSessionId, serverSeed, config
3. Client generates clientSeed, computes finalSeed
4. Questions selected via seeded RNG from local question bank (30 items)
5. For each question:
   a. Flash phase: Question displayed for flashDurationMs
   b. Type phase: Player types answer, has answerTimeMs to respond
   c. Auto-submit on timeout (empty answer = wrong)
   d. Score: pointsPerCorrect + speedBonus (based on response time)
   e. 1500ms result transition
6. After all questions: finished phase, ResultScreen

Scoring:
- pointsPerCorrect: base points for correct answer
- speedBonusMax: max bonus scaled by response speed
- speedRatio = max(0, 1 - responseTimeMs / answerTimeMs)
- bonus = round(speedRatio × speedBonusMax)
- Case-insensitive comparison, trimmed

================================================================================
4. GAME CONFIG (from server)
================================================================================

GameConfig interface:
- flashDurationMs: how long question is shown (ms)
- answerTimeMs: time limit to type answer (ms)
- totalQuestions: number of questions per game
- pointsPerCorrect: base score per correct answer
- speedBonusMax: maximum speed bonus

================================================================================
5. QUESTION BANK
================================================================================

30 trivia questions hardcoded in questionBank.ts:
- General knowledge (capitals, science, math, geography)
- Short answers (1-2 words or numbers)
- Selected via seeded RNG (mulberry32)
- Seed: computeFinalSeed(serverSeed, clientSeed, nonce=0)

Examples:
- "Capital of France?" → "paris"
- "Chemical symbol for gold?" → "au"
- "Square root of 144?" → "12"

================================================================================
6. API INTEGRATION
================================================================================

httpClient: axios with encryption interceptors
- Request: Bearer token from localStorage, payload encryption
- Response: payload decryption

Endpoints:
- POST /game/start { clientSeed } → { gameSessionId, serverSeed, config, serverTime }
- POST /game/submit { gameSessionId, answers, clientScore } → { finalScore, status, correctAnswers }

Answer type: { questionId, userAnswer, timestamp, responseTimeMs }
